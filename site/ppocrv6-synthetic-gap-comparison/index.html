第一轮 ABO 合成训练结果：YOLO11m-OBB vs PP-OCRv6

只包含 2026-06-25 第一轮 ABO/真实商品背景合成数据训练结果，不混入后续 Business6k 统一评估、V2 或继续训练结果。

一句话结论

第一轮训练证明 YOLO OBB 和 PP-OCRv6 都能在合成数据上收敛，但迁移到真实业务 benchmark 后 recall 都很低。YOLO test recall 只有 36.70%，PP-OCRv6 test recall 只有 49.30%；两者都不能满足 overlay 擦除“不漏检”的业务要求。

YOLO11m-OBB召回更低，但 clean/native 误检低于 OCR，可作为第一轮主 baseline。
PP-OCRv6文字 overlay recall 更高，但 clean/native FP 很严重，误擦商品本体风险更高。
核心问题ABO 合成数据与真实业务视频存在 domain gap，合成 val 高分不能代表业务效果。

判断与下一步

  • 第一轮 ABO 合成数据能训练出模型，但不能覆盖真实业务 overlay 分布；YOLO 合成 val mAP50-95 约 98.86%，业务 test mAP50-95 只有 8.60%。
  • PP-OCRv6 在文字 overlay 上 recall 更高，但 clean/native FP 明显高于 YOLO，说明 OCR 先验会把包装字、logo、屏幕 UI、native text 当成 overlay。
  • 两条路线都未达到可上线标准；下一轮应优先重构数据，而不是继续在当前 ABO 合成集上加 epoch。
  • 建议下一版数据以字幕/水印为主：subtitle 约 80%，watermark 12%-15%，emoji/贴纸/UI/价格条/符号低比例保留，并加入 hard negative。

汇报口径

第一轮实验结论是：YOLO11m-OBB 和 PP-OCRv6 都可以作为 overlay 检测 baseline，但 ABO 合成数据到真实业务 benchmark 的泛化不足。YOLO 的 clean/native 误检低于 OCR，适合作为主 baseline；OCR 的文字召回更好，但误擦风险更高。下一阶段重点是构建更贴近真实业务的 subtitle/watermark 数据集，并加入 hard negative 和 clean/native 误检评估。
